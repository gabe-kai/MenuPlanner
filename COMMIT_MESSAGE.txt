Phase 3.9 – Admin Console & Cross-Entity Management

Implemented full phase 3.9 completion for admin directory sync and mutating admin actions.

- Added real-auth admin directory API and auth-guarded actor checks.
  - New `GET /api/admin/directory` route with centralized actor validation.
  - Directory payload now includes authoritative users, families, edit policies, and password-reset flags.
- Extended auth identity services (`src/lib/auth/authIdentity.server.ts`) for admin operations:
  - Added admin identity exports: `listAuthIdentities`, `listAuthFamilies`.
  - Added family metadata/state helpers and persistence helpers for family naming.
  - Added `createAuthFamily`, `setAuthFamilyName`, and `setAuthIdentityFamily`.
  - Added `setAuthIdentityTemporaryPassword`, `setPasswordChangeRequired`, and `deleteAuthIdentity`.
  - Kept password update semantics aligned so admin-initiated resets always set `mustChangePassword`.
- Implemented admin-user management routes:
  - `POST /api/admin/users/[userId]/reset` to force password-reset behavior and optional temporary password.
  - `DELETE /api/admin/users/[userId]` to delete identities (self-delete blocked).
  - `PATCH /api/admin/users/[userId]/family` to move users across families.
- Implemented admin-family management routes:
  - `POST /api/admin/families` to create families.
  - `PATCH /api/admin/families/[familyId]` to rename families.
- Updated admin UI pages to consume real-auth admin directory and persist actions:
  - `src/app/admin/users/page.tsx`
    - Added directory sync via `/api/admin/directory`.
    - Added password-reset status column and reset/delete buttons.
    - Added API-backed reset/delete flows in real-auth mode.
  - `src/app/admin/families/page.tsx`
    - Added directory sync via `/api/admin/directory`.
    - Added real-auth create/rename/bootstrap/seed flows for families.
    - Added real-auth move flows for user migration from `fam-1`.
- Added admin mutation audit logs for all new admin actions in logs module.
- Updated implementation plan completion notes to reflect new 3.9 work.
